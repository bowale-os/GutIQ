backend/app/rag/retriever.py
Pain relief pipeline — red-flag detection, condition inference, Qdrant search
▶
fn retrieve(request) → RetrievalResult
Main entry point. Red flag check → condition inference → Qdrant search → return result.
fn _check_red_flags(request)
13 keyword patterns (blood, vomiting, fever, etc.) + structural rule: lower_right region + intensity ≥ 6 → appendicitis flag. Returns (is_red_flag, reason).
fn _resolve_condition(body_clicks, description, pain_character)
Scoring algorithm: region → primary 2.0 / secondary 1.0 points, keyword match → +1.5, pain character → +1.0. Returns highest-scoring GutCondition.
fn _search(condition, query)
Qdrant similarity search filtered by condition. If <2 results → falls back to unfiltered search. Returns top-K chunks.
Collection: "gut_pain_relief" · Embedding: BAAI/bge-small-en-v1.5 (384-dim) · TOP_K: 6 chunks

ETL pipeline — PubMed + PDFs → Parquet cache → Qdrant Cloud
▶
fn _fetch_pubmed()
Queries PubMed via LangChain with 100+ queries across 11 conditions. Checkpoints every 10 queries. Skips PMIDs already in cache (resumable).
fn _load_pdfs()
Glob knowledge/pdfs/**/*.pdf. Infers condition tag from parent subfolder name via PDF_FOLDER_MAP.
fn _split(records)
RecursiveCharacterTextSplitter: 512 chunk size, 64 overlap. Preserves condition metadata per chunk.
fn _upsert_to_qdrant(chunks, recreate)
Batch embed + upsert. Batch size 100. Creates payload index on condition field for filtered search.
fn _count_pubmed()
Dry-run. Counts how many abstracts each condition query would return without ingesting anything. Run with --count-only.
CLI flags: --from-cache skip PubMed fetch · --recreate drop + rebuild Qdrant collection · --count-only dry-run counts
PDF folder structure: knowledge/pdfs/<ConditionName>/file.pdf — folder name maps to condition via PDF_FOLDER_MAP
